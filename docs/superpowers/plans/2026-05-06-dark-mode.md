# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a persistent dark mode toggle to the masthead that respects OS preference by default and stores manual overrides in `localStorage`.

**Architecture:** Dark mode colors live in a `[data-theme="dark"]` block that overrides five CSS custom properties on `:root`. A tiny inline script in `<head>` sets the attribute before paint to prevent flash. JS in `main.js` wires up the button.

**Tech Stack:** Vanilla CSS custom properties, vanilla JS, Jekyll (Liquid templates)

### Task 3: Add no-FOUC inline script to default.html

**Files:**
- Modify: `_layouts/default.html`

This script must run synchronously before any CSS is applied, so it goes inside `<head>` before the closing `</head>` tag.

- [ ] **Step 1: Add the inline script**

Open `_layouts/default.html`. Add the following block immediately before `</head>`:

```html
  <script>
    (function () {
      var stored = localStorage.getItem('theme');
      if (stored === 'dark' || (!stored && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
        document.documentElement.setAttribute('data-theme', 'dark');
      }
    })();
  </script>
```

The full `<head>` should end like this:

```html
  <link href="..." rel="stylesheet">
  <script>

        document.documentElement.setAttribute('data-theme', 'dark');
      }
    })();
  </script>
</head>
```

### Task 4: Add JS toggle logic to main.js

**Files:**
- Modify: `js/main.js`

- [ ] **Step 1: Add theme toggle logic**

Open `js/main.js`. Replace the entire file with:

```javascript
document.addEventListener('DOMContentLoaded', function () {
  // Mobile nav toggle
  const navToggle = document.getElementById('nav-toggle');
  const nav = document.getElementById('main-nav');

  if (navToggle && nav) {
    navToggle.addEventListener('click', function () {
      const isOpen = nav.classList.toggle('open');
      navToggle.setAttribute('aria-expanded', String(isOpen));
      navToggle.textContent = isOpen ? 'Close' : 'Menu';
    });
  }

  // Theme toggle
  const themeToggle = document.getElementById('theme-toggle');
  if (themeToggle) {
    const html = document.documentElement;

    function updateLabel() {
      themeToggle.textContent = html.getAttribute('data-theme') === 'dark' ? 'LIGHT' : 'DARK';
    }

    updateLabel();

    themeToggle.addEventListener('click', function () {
      const isDark = html.getAttribute('data-theme') === 'dark';
      if (isDark) {
        html.removeAttribute('data-theme');
        localStorage.setItem('theme', 'light');
      } else {
        html.setAttribute('data-theme', 'dark');
        localStorage.setItem('theme', 'dark');
      }
      updateLabel();
    });
  }
});
```

- [ ] **Step 2: Verify in browser**

Serve the site locally (`bundle exec jekyll serve`) and open `http://localhost:4000`.

Check:
- Button shows `DARK` on initial load (light mode)
- Clicking switches to dark mode — background goes from cream to charcoal, text goes from near-black to warm off-white
- Button now shows `LIGHT`
- Clicking again restores light mode
- Reload the page — dark mode persists (button shows `LIGHT`, dark colors active)
- Open DevTools → Application → Local Storage: `theme` key should be `"dark"` or `"light"`
- To test system preference: clear localStorage, set OS to dark mode — site should start in dark mode automatically
